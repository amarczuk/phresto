Server side

There are several versions of PHP available and you can pick the one you want to use from run options menu.

You can use any function available in the PHP version of choice and most of the popular extensions. For the full list of extensions just execute phpinfo();.

You won't be able to use Apache specific functions (as it's not Apache server).

Use include or require to include other scripts from the sandbox. You can read any file existing in the sandbox or upload files using html form (just put the name of your php script in action attribute). None of the saved files will be available to read next time you execute script

You can use python scripts executed via http request (as CGI module). Just make sure that the first thing you print is http header.

Use import to include python scripts from the sandbox (you don't need to create __init__.py - just use the file name).

Make request from html page using any type of http request (that includes posting files).

Create perl scripts and packages. Just use your package file name with use or perl script name with required.

Perl scripts are executed via http (using CGI) and the first printed line should be a valid http header.

Check the list of modules you can use here

Client side

Create html page to test your JS or ajax request to the backend (php, python, perl). If you won't add html tag, doctype and headers this will be auto generated based on the run options.

You can link any file on the sandbox or use it as a form action target. You can attach css or js files created on your sandbox as well.

You can execute javascript files directly and they will be wrapped in html. You can specify additional options from the run menu.

create css files to use them on html pages on your sandbox. You won't be able to execute the css file on its own but it's just a few clicks to create test html page as well.

Data source

Add xml, json, csv or txt files to pass additional data to the scripts you're testing. You can access them by the file name both on the server and client side
