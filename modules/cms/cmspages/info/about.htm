Sandbox

The idea is simple. If you want to test something quickly, just get the new sandbox, add files and run code. You can also download your work and use in your projects.

Don't need to clean after yourself - it will disappear if not used for more than a week (one-off) or a month (owned one). But don't worry - by used we mean opened on PHPfy or as embedded sample on your website.

How my code gets executed?

Your code is placed on isolated server and accessed via http, then result of the execution is printed to the output pane.

Every execution creates it's own environment so you should not relay on any files created by your script (but you can send files to the script in a POST request and you can access any file created on the sandbox). We don't block any functions. However there is 30s maximum execution time limit.

For more details go to the help section.

Little bit more

Having own sandbox means only you can modify code. But you can give edit permission to the other PHPfy users creating a group responsible for maintaining the sandbox. Owned sandbox may be also blocked to the general public so only editors can access it.

You can create the embeddable version of your code just by picking the files and selecting the main one to be run. We'll give you a piece of JS to add to your page.

Create new or clone existing sandbox in seconds

If you like to play with the code without changing original sandbox simply press clone and you'll have a full replica in a few seconds time.

Enyone can clone the sandbox from the sample embedded on your page. The only difference is that only files included in your example will be copied over. That means you can use same sandbox for creating different samples.

Leave your comments

You can have a discussion around the sandbox's content. Just navigate to the comments section in sandbox settings menu.

May I break it?

We hope you can't. But if you do - please let us know so we can make it safe again.
